01 BlueK

Summary

During my first year at the University of Waterloo, my team and I were assigned a culminating project, requiring us to build a robot that could perform a task of our choice, integrating both mechanical and software components. Our final project, named BlueK, focused on developing a robot with the ability to identify color pairs on a 3x4 board.

Mechanical Overview

The robotic structure was assembled using LEGO elements, with the LEGO Mindstorm EV3 serving as the central processing unit. The 3x4 board, which contained the color pairs, was also constructed with LEGO components. Additionally, the framework that provided the robot's movement was made using aluminum extrusions. As the picture is shown above, the robot navigates the two-dimensional space to access each card arranged in a table format.

Software Overview

The language we used to develop the color solving logic was ROBOTC, which is a text-based programming language based on the standard C programming language. There were constraints due to the accuracy of the robot’s sensors, motor encoder, and limited materials. To overcome these constraints, we had BlueK flip two cards in a row each time. If the two adjacent cards were the same color, it would leave it flipped over. However, if they were different, BlueK would just flip it over back to its uncolored side and store the position of the color in an array. After it flips all 12 cards, it would proceed to loop through the array and find the first color that has not been paired yet.
